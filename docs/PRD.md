# 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目名称
Auto-Workflow-Test（自动化工作流测试项目）

### 1.2 项目背景
本项目是一个轻量级 C++ 练手项目，主要用于学习和测试基础编程概念、Git 版本控制工作流以及 Claude Code 自动化辅助开发流程。项目旨在通过简单的命令行程序演示基本的输入输出处理和逻辑判断。

### 1.3 项目定位
面向编程初学者和开发流程测试场景的极简命令行工具集。

---

## 2. 项目目标

### 2.1 核心目标
- 实现一套基础数学工具命令行程序（奇偶判断、闰年判断、加法计算等）
- 建立规范的 Git 工作流和项目结构
- 集成 GitHub Actions CI 自动化编译与测试
- 集成 GitHub Issue 驱动的需求自动录入流程
- 集成 Claude Code 自动轮询实现（loop 每 2 分钟检查 newquest.md）
- 集成 Claude Code 辅助开发配置（hooks 自动编译验证）

### 2.2 扩展目标
- 后续可扩展更多基础算法演示程序
- 可作为 CI/CD 流程、代码审查等开发实践的测试载体
- 探索 AI 辅助编程在简单项目中的最佳实践

用户启动程序 → 提示"请输入两个整数:" → 用户输入两个数字

## 5. 技术架构

### 5.1 项目结构

```
/
├── leap_year.cpp         # 闰年判断程序
├── add.cpp               # 两数相加程序
├── hello.cpp             # Hello World 程序
├── love_mom.cpp          # 输出"我爱妈妈"程序
├── mul.cpp               # 两数相乘程序
├── .github/
│   └── workflows/
│       ├── ci.yml             # GitHub Actions CI 配置（编译 + 测试）
│       └── issue-driver.yml   # Issue 驱动：创建 Issue 自动写入 newquest.md
├── .gitignore            # Git 忽略规则
├── CLAUDE.md             # Claude Code 项目指引
├── .claude/
│   └── settings.local.json  # Claude Code 本地配置
└── docs/
    ├── PRD.md            # 本需求文档
    └── newquest.md       # 待定文档（预留）
```

### 5.2 技术栈

- **语言**: C++
- **构建**: 直接使用 g++ 编译（`g++ test.cpp -o test`）
- **版本控制**: Git + GitHub 远程仓库
- **辅助工具**: Claude Code（权限配置、开发辅助）

### 5.3 核心代码说明

**leap_year.cpp** — 闰年判断：
1. 声明整型变量 `year`
2. 输出提示信息 `请输入一个年份:`
3. 从标准输入读取整数
4. 通过 `(year % 400 == 0) || (year % 4 == 0 && year % 100 != 0)` 判断闰年
5. 输出中文结果

**add.cpp** — 两数相加：
1. 声明整型变量 `num1`, `num2`
2. 输出提示信息 `请输入两个整数:`
3. 从标准输入读取两个整数
4. 计算 `sum = num1 + num2`
5. 输出算式和结果

**hello.cpp** — Hello World：
1. 输出字符串 `Hello World`
2. 程序结束

**love_mom.cpp** — 输出"我爱妈妈"：
1. 输出字符串 `我爱妈妈`
2. 程序结束

**mul.cpp** — 两数相乘：
1. 声明整型变量 `num1`, `num2`
2. 输出提示信息 `请输入两个整数:`
3. 从标准输入读取两个整数
4. 计算 `product = num1 * num2`
5. 输出算式和结果

## 8. 开发与部署

### 8.1 本地运行

```bash
# 闰年判断
g++ leap_year.cpp -o leap_year && ./leap_year

# 两数相加
g++ add.cpp -o add && ./add

# Hello World
g++ hello.cpp -o hello && ./hello

# 输出"我爱妈妈"
g++ love_mom.cpp -o love_mom && ./love_mom

# 两数相乘
g++ mul.cpp -o mul && ./mul
```

### 8.2 版本管理

- 分支策略：GitHub Flow（主分支 `main` + 功能分支）
- 提交规范：语义化提交信息

### 8.3 自动化工作流

项目实现了两层自动化推进机制：

**① GitHub Issue 驱动（触发层）**
```
创建 Issue → issue-driver.yml 触发
  → 将 Issue 标题和内容写入 docs/newquest.md
  → commit + push 到 main 分支
```

**② Claude Code Loop（执行层）**
```
每 2 分钟检查 docs/newquest.md
  → 有需求：创建/编辑/删除 .cpp 文件 → 编译 → 更新 PRD.md → 清空 newquest.md
  → 无需求：等待下次检查
```

### 8.4 Claude Code 集成

项目配置了 Claude Code 权限白名单，支持自动化执行：
- `Skill(update-config)` — 配置管理
- `Bash(git *)` — Git 操作
- `Bash(git commit *)` — 提交代码
- `Bash(git push *)` — 推送代码
